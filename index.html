Questionário 1

Questão 1: Função de um servidor web em um ambiente de hospedagem + exemplo.

Resposta: Um servidor web recebe e processa requisições HTTP/HTTPS de clientes (geralmente navegadores), entregando conteúdos como páginas HTML, imagens e scripts. Exemplo: Apache HTTP Server.
Questão 2: Principal característica do NGINX e por que é muito utilizado

Resposta: O NGINX é conhecido por sua alta performance e uso eficiente de recursos, especialmente para lidar com muitas conexões simultâneas. É amplamente utilizado em projetos de alto desempenho por sua arquitetura assíncrona e escalabilidade.
Questão 3: Diferença entre HTTP e HTTPS + motivo para uso do HTTPS

Resposta: • HTTP: protocolo sem criptografia.
• HTTPS: versão segura do HTTP, com criptografia TLS/SSL.
O HTTPS é recomendado porque protege os dados contra interceptações e ataques (como man-in-the-middle), essencial para segurança e confiança do usuário.
Questão 4: Função das portas e importância da porta 443

Resposta: As portas identificam serviços específicos em um endereço IP.
A porta 443 é usada para conexões HTTPS e é essencial para a segurança da comunicação na web.
Questão 5: Endereço IP e sua importância

Resposta: O IP (Internet Protocol) é o identificador único de um dispositivo em uma rede.
É essencial para rotear dados corretamente entre dispositivos em redes locais ou na internet.
Questão 6: O que é hospedagem VPS e quando usá-la

Resposta: VPS (Virtual Private Server) é uma máquina virtual com recursos dedicados em um servidor físico compartilhado.
Indicada quando se precisa de mais controle, desempenho e segurança que a hospedagem compartilhada, sem o custo de um servidor dedicado.
Questão 7: Vantagens do Caddy

Resposta: • HTTPS automático com Let’s Encrypt
• Configuração simples via arquivos de configuração minimalistas
• Suporte nativo a HTTP/2 e HTTP/3
Ideal para desenvolvedores que querem segurança e praticidade.
Questão 8: Servidor mais indicado para aplicações .NET

Resposta: O Microsoft IIS (Internet Information Services) é o mais indicado, pois é otimizado para aplicações ASP.NET e integração com o ecossistema Microsoft (como Windows Server).
Questão 9: Modelo Cliente-Servidor em sistemas web

Resposta: O cliente (ex: navegador) envia uma requisição ao servidor, que processa e responde com os dados (como páginas ou APIs).
O servidor centraliza recursos e lógica, enquanto o cliente consome os serviços.
Questão 10: O que é o Apache Tomcat e para que é indicado

Resposta: É um servidor de aplicação Java focado em executar aplicações web baseadas em servlets e JSP. Ideal para aplicações web em Java EE (agora Jakarta EE).
Questão 11: Servidor web vs. Servidor de aplicação

Resposta: • Servidor web: entrega conteúdo estático (HTML, CSS).
• Servidor de aplicação: processa lógica de negócio e executa código (como Java, .NET).
Ambos podem trabalhar juntos.
Questão 12: Conceito de domínio e sua relação com IP

Resposta: Um domínio é um nome legível (ex: google.com) que aponta para um endereço IP. O DNS faz essa tradução para facilitar o acesso humano.
Questão 13: Vantagens da hospedagem em nuvem vs. local

Resposta: • Escalabilidade sob demanda
• Alta disponibilidade
• Backup e recuperação mais fáceis
• Menor custo inicial
Hospedagem local exige infraestrutura e manutenção.
Questão 14: Exemplos de servidores de aplicação

Resposta: • Tomcat – Java
• Gunicorn – Python (WSGI)
Questão 15: Importância do HTTPS atualmente

Resposta: HTTPS é essencial para segurança, privacidade e confiança.
HTTP expõe dados a interceptações, comprometendo senhas, dados bancários e sessões.
Questão 16: Exemplo de uso de VPS

Resposta: Uma loja virtual de médio porte com CMS (como WordPress com WooCommerce) que precisa de mais recursos e personalização do que a hospedagem compartilhada pode oferecer.
Questão 17: Diferença entre IP e porta

Resposta: • IP: identifica o host.
• Porta: identifica o serviço rodando no host.
Juntos, definem uma conexão única (ex: 192.168.1.1:443).
Questão 18: Importância do suporte a HTTPS em servidores

Resposta: Sem HTTPS, as conexões ficam vulneráveis. Um servidor web que suporta HTTPS protege dados sensíveis e integridade da comunicação, essencial para privacidade.
Questão 19: Dois exemplos de servidores web

Resposta: • Apache HTTP Server: extensível, compatível com muitos módulos.
• NGINX: leve, excelente para alta carga, reverso proxy.
Questão 20: Hospedagem compartilhada

Resposta: Múltiplos sites em um único servidor.
Indicada para sites pequenos ou iniciantes com baixo tráfego, devido ao baixo custo.
Questão 21: Função do servidor de aplicação

Resposta: Executa lógica de negócio e gera conteúdo dinâmico (ex: consultas ao banco).
Diferente de servidores web que servem apenas arquivos estáticos.
Questão 22: Servidor de aplicação para Python

Resposta: Gunicorn: servidor WSGI usado com frameworks como Django ou Flask.
Recebe requisições e as encaminha para a aplicação Python via WSGI.
Questão 23: WildFly (ex-JBoss)

Resposta: Servidor de aplicação para Java EE (Jakarta EE).
Utilizado em aplicações corporativas Java com suporte completo a especificações enterprise.
Questão 24: Função da porta 21

Resposta: Usada pelo protocolo FTP (File Transfer Protocol) para transferência de arquivos.
Importante para uploads e downloads em servidores.
Questão 25: LiteSpeed e compatibilidade com Apache

Resposta: LiteSpeed é compatível com arquivos .htaccess, configurações e módulos do Apache.
Vantagens: melhor desempenho, menor uso de memória, suporte nativo a HTTP/3 e segurança avançada.
Questão 26: O que é o Microsoft IIS

Resposta: Servidor web da Microsoft.
Usado em ambientes corporativos que utilizam Windows Server e aplicações ASP.NET.
Questão 27: Função do Node.js

Resposta: Ambiente de execução para JavaScript no servidor.
Usado para aplicações web em tempo real, APIs e sistemas de alta concorrência.
Questão 28: Função do GlassFish

Resposta: Servidor de aplicação para Java EE.
Executa EJBs, servlets, JSF, entre outros, indicado para grandes aplicações corporativas.
Questão 29: Puma e Unicorn

Resposta: São servidores de aplicação para Ruby (ex: Rails).
• Puma: concorrência por threads, ideal para apps modernas.
• Unicorn: multiprocessos, estável para ambientes com carga previsível.
Questão 30: Importância da escolha correta da hospedagem

Resposta: A escolha correta (local, compartilhada, VPS, nuvem) impacta diretamente em:
• Desempenho
• Escalabilidade
• Custo
• Segurança
Cada tipo atende diferentes níveis de complexidade e demanda do projeto.
